# Troubleshooting

Solutions to common issues running your Hytale server.

## Server Won't Start

### Container Exits Immediately

**Check logs for the error:**

```bash
docker compose logs hytale-server
```

**Common causes:**

1. **Java version too old** - Image includes Java 25+, but if building custom image, ensure Java 25+
2. **hytale-downloader missing** - Verify the Dockerfile includes hytale-downloader
3. **Download failed** - Network issues or auth problems with hytale-downloader

### Download Fails

**Symptoms:**
- "Download failed" in logs
- "No zip file found"

**Solutions:**

1. Check network connectivity:
   ```bash
   docker compose exec hytale-server ping -c 4 google.com
   ```

2. Re-run with fresh download:
   ```bash
   docker compose down
   FORCE_UPDATE=true docker compose up -d
   ```

3. Check hytale-downloader authentication (may need to re-auth the downloader itself)

## Authentication Issues

### OAuth Device Code Not Appearing

**Symptoms:**
- Server starts but no device auth URL appears
- Download hangs without auth prompt

**Verify logs:**

```bash
docker compose logs | grep -i auth
```

### Tokens Not Persisting (Re-auth Every Restart)

**Symptoms:**
- Device auth required on every restart
- Tokens expire or disappear

**Solution - Ensure hytale-data volume is persisted:**

Tokens are stored in `/server/.hytale/tokens/` which is part of the `hytale-data` volume.

**Check token status:**

```bash
docker exec hytale-server hytale-auth status
docker exec hytale-server ls -la /server/.hytale/tokens/
```

### Device Code Expires

Device codes are valid for 15 minutes. If expired:

```bash
docker compose restart
docker compose logs -f
```

Complete authorization faster this time.

### Can't Access Auth URL

1. Check internet connectivity
2. Try accessing URL from a different device
3. Verify no firewall blocking Hytale domains

### Profile Selection Required

**Symptoms:**
- Server waits at "Waiting for profile selection..."
- Multiple profiles on your Hytale account

**Solution:**

```bash
# List available profiles
docker exec -it hytale-server hytale-auth profile list

# Select a profile (by number)
docker exec -it hytale-server hytale-auth profile select 1
```

Or enable auto-selection:

```yaml
environment:
  AUTOSELECT_GAME_PROFILE: "true"
```

## Console/Command Issues

### Commands Not Working

**Symptoms:**
- `hytale-cmd` returns no output
- Commands don't seem to execute

**Solutions:**

1. **Check server is running:**
   ```bash
   docker compose ps
   ```

2. **Check server logs for errors:**
   ```bash
   docker compose logs --tail=50
   ```

3. **Ensure stdin/tty are enabled in docker-compose.yml:**
   ```yaml
   services:
     hytale:
       stdin_open: true
       tty: true
   ```

## Connection Issues

### Players Can't Connect

**Checklist:**

1. **Server is running:**
   ```bash
   docker compose ps
   ```
   
2. **Port is UDP:**
   ```yaml
   ports:
     - "5520:5520/udp"  # Must have /udp!
   ```

3. **Server is authenticated:**
   ```bash
   docker exec hytale-server hytale-auth status
   ```

4. **Firewall allows UDP 5520:**
   ```bash
   # Linux
   sudo ufw allow 5520/udp
   
   # Windows
   netsh advfirewall firewall add rule name="Hytale" dir=in action=allow protocol=UDP localport=5520
   ```

5. **Router port forwarding configured for UDP**

### Connection Timeout

**Causes:**
- Firewall blocking UDP
- Wrong protocol (TCP instead of UDP)
- Router not forwarding
- ISP blocking UDP

**Debug steps:**

1. Test locally first: `localhost:5520`
2. Test on LAN: `192.168.x.x:5520`
3. Test externally with correct public IP

## Performance Issues

### Out of Memory

**Symptoms:**
- `java.lang.OutOfMemoryError`
- Container crashes/restarts

**Increase memory:**

```yaml
environment:
  JAVA_OPTS: "-Xms6G -Xmx8G -XX:+UseG1GC"

deploy:
  resources:
    limits:
      memory: 10G
```

### Slow Startup

1. **Enable AOT cache:**
   ```yaml
   environment:
     USE_AOT_CACHE: "true"
   ```

2. **Use SSD storage**

3. **First startup is always slower** (downloads + AOT cache generation)

### Server Lag

1. **Check resource usage:**
   ```bash
   docker stats hytale-server
   ```

2. **Increase memory allocation**

3. **Check host system resources**

## Volume Issues

### Permission Denied

**Symptoms:**
- Can't write to server files
- "Permission denied" in logs

**Fix permissions:**

```bash
docker compose down

# Find volume path
docker volume inspect hytale-docker_hytale-data | grep Mountpoint

# Fix ownership (container runs as UID 1000)
sudo chown -R 1000:1000 /path/to/volume

docker compose up -d
```

### Data Lost After Restart

**Ensure volume is defined:**

```yaml
volumes:
  hytale-data:
```

**Check volume exists:**

```bash
docker volume ls | grep hytale
```

## Update Issues

### Server Won't Update

**Force re-download:**

```bash
docker compose down
FORCE_UPDATE=true docker compose up -d
```

**Or rebuild image:**

```bash
docker compose down
docker compose build --no-cache
docker compose up -d
```

### Check Current Version

The downloader version is saved to `/server/.downloader_version`:

```bash
docker compose exec hytale-server cat /server/.downloader_version
```

## Docker Issues

### Image Build Fails

```bash
# Clean rebuild
docker compose build --no-cache

# Or pull latest
docker compose pull
```

### Container Won't Stop

```bash
# Force stop
docker compose kill

# Remove
docker compose rm -f
```

## Advanced Debugging

### Access Container Shell

```bash
docker compose exec hytale-server bash
```

Inside:
```bash
# Check files
ls -la /server/

# Check processes
ps aux

# Check logs
cat /tmp/server_output.log
```

### Export Full Logs

```bash
docker compose logs > hytale-debug.log
```

### Watch Real-time Output

```bash
docker compose logs -f --tail=100
```
